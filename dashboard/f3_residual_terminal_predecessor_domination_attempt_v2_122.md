## Exact no-closure blocker

The interface currently quantifies over every pair

```lean
residual essential : Finset (ConcretePlaquette physicalClayDimension L)
```

assuming only:

```lean
essential ⊆ residual
```

For `residual.card ≠ 1`, it then requires every essential parent `p` to have a
selected terminal predecessor `q ∈ residual` such that:

```lean
p.1 ∈ (plaquetteGraph physicalClayDimension L).neighborFinset q.1
```

This is not implied by `essential ⊆ residual`.  A minimal obstruction is a
two-point residual bucket

```text
residual = {p, q}
essential = {p}
```

where `p` and `q` are not adjacent in `plaquetteGraph`.  Since the graph is
loopless, `p` cannot select itself as its terminal predecessor, and since `q`
is not adjacent to `p`, no residual terminal predecessor can satisfy the
last-edge domination clause.

In the concrete physical graph this pattern corresponds to two physical
plaquettes with base sites at site-lattice distance greater than `1`; the
current residual-only interface does not require the residual to be connected,
anchored, fiber-produced, or locally dominating around `essential`.

## Why this is not just a missing proof

The failure is structural: the theorem asks for last-edge domination from data
that does not include any last-edge adjacency hypothesis.  Existing APIs supply
such adjacency only in more restricted contexts:

- current safe-deletion witnesses `(X, deleted X)`;
- anchored/preconnected residual fibers generated by base-aware bookkeeping;
- local neighbor shells of a fixed portal or parent.

Using those APIs directly here would violate the task stop condition, because
it would choose terminal predecessors from the current deleted-vertex witness
instead of residual-only data.
